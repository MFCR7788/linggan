# 灵集 - 配置说明文档

## ⚠️ 重要安全提示

**不要将包含真实API Key的 .env.local 文件提交到Git仓库！**

已配置的 `.gitignore` 文件会自动保护敏感文件，但请务必确认：
- [ ] `.env.local` 文件没有被加入Git暂存区
- [ ] 所有API Key都只在本地环境文件中
- [ ] 生产环境使用环境变量管理，不要硬编码

---

## 📁 文件说明

| 文件 | 用途 | 是否提交Git |
|------|------|------------|
| `.env.example` | 配置模板，包含所有可用配置项（不包含真实密钥） | ✅ 是 |
| `.env.local` | 本地开发环境的真实配置（需要你从模板复制） | ❌ 否 |
| `.gitignore` | Git忽略文件配置 | ✅ 是 |

---

## 🚀 快速开始

### 1. 创建本地配置文件

```bash
cd /Users/aplle/Documents/Zjsifan/Tools/linggan
cp .env.example .env.local
```

### 2. 完善配置（可选）

打开 `.env.local` 文件，按需填写以下缺失配置：
- `SUPABASE_SERVICE_ROLE_KEY`
- `ALIYUN_OSS_ACCESS_KEY_ID`
- `ALIYUN_OSS_ACCESS_KEY_SECRET`
- `ALIYUN_OSS_BUCKET`
- `ALIYUN_OSS_REGION`
- `ALIYUN_STT_ACCESS_KEY_ID`
- `ALIYUN_STT_ACCESS_KEY_SECRET`

### 3. 开始开发

配置完成后即可开始项目开发！

## 🔍 关于 Supabase Auth 的用途

Supabase Auth 是用户认证系统，用于：

| 功能 | 说明 |
|------|------|
| 用户注册/登录 | 支持手机验证码、微信登录、邮箱登录 |
| 会话管理 | 自动管理用户登录状态和刷新 Token |
| 权限控制 | 控制用户只能访问自己的数据（灵感、热点等） |
| 安全验证 | 与 Supabase Database/Storage 集成，确保数据安全 |

简单说：**Auth 负责让用户登录，确保只有登录的人才能看到自己的内容。**

## 🔑 主要配置项说明

### AI相关
| 变量 | 说明 |
|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek文本生成API密钥 |
| `DOUBAO_API_KEY` | 火山引擎豆包多模态API密钥 |
| `SEEDANCE_VIDEO_MODEL_ARK_ID` | 视频生成模型ID |
| `SEEDANCE_IMAGE_MODEL_ARK_ID` | 图片生成模型ID |
| `OPENROUTER_API_KEY` | OpenRouter多模型API密钥 |
| `ELEVENLABS_API_KEY` | ElevenLabs语音合成API密钥 |

### 数据库和认证
| 变量 | 说明 |
|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase项目URL |
| `SUPABASE_ANON_KEY` | Supabase公开访问密钥 |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase服务端密钥（可选） |
| `DATABASE_URL` | PostgreSQL数据库连接字符串 |

### 阿里云服务
| 变量 | 说明 |
|------|------|
| `ALIYUN_OSS_*` | 对象存储相关配置 |
| `ALIYUN_STT_*` | 语音识别相关配置 |
| `ALIYUN_SMS_*` | 短信服务相关配置（已完善） |
