## 🎨 设计规范统一前置

**所有页面共用（玻璃拟态科技风格 + 模块化设计）：**

### 1. 色彩系统
| 色彩类型 | 颜色值 | 用途 |
|---------|-------|------|
| 背景渐变 | 从 `#0A1629` 到 `#1A365D` | 页面整体背景 |
| 主色 | `#3B82F6` | 激活按钮、高亮数据、选中标签，带轻微霓虹发光效果 |
| 辅助色 - 紧急 | `#EF4444` | 错误、警告、紧急状态，带轻微柔光 |
| 辅助色 - 成功 | `#22C55E` | 成功、正常状态，带轻微柔光 |
| 玻璃卡片背景 | `rgba(255, 255, 255, 0.12)` | 半透明玻璃效果 |
| 玻璃描边 | `rgba(255, 255, 255, 0.3)` | 1px 细白色描边 |
| 标题文字 | `#FFFFFF` | 所有标题文字 |
| 正文文字 | `#E5E7EB` | 所有正文文字 |
| 次要文字 | `#9CA3AF` | 辅助信息、占位符 |

### 2. 玻璃拟态效果
- **毛玻璃模糊**：8-10px 背景模糊
- **圆角**：16px（卡片）、8px（标签、按钮）
- **阴影**：轻微柔光投影（`0 4px 6px rgba(0, 0, 0, 0.1)`）
- **发光效果**：主色按钮添加 `0 0 20px rgba(59, 130, 246, 0.5)` 外发光
- **Hover 效果**：玻璃反光效果，透明度稍微加深

### 3. 文字规范
- **字体**：现代无衬线字体（推荐 Inter 或 PingFang SC）
- **字重**：标题 600，正文 400
- **字号层级**：
  - 大标题：28px
  - 中标题：20px
  - 小标题：16px
  - 正文：14px
  - 辅助文字：12px

### 4. 间距规范
- **小间距**：8px
- **中间距**：16px
- **大间距**：24px
- **超大间距**：32px

### 5. 图标规范
- **风格**：极简线性风格（2px 描边）
- **大小**：24px（主要图标）、20px（次要图标）
- **颜色**：与文字颜色一致（`#FFFFFF` 或 `#E5E7EB`）
- **外发光**：轻微外发光（`0 0 10px rgba(255, 255, 255, 0.2)`）

### 6. 星空粒子纹理
- 在 Figma 中创建一个单独的图层作为背景
- 使用低透明度（15-20%）的白色圆点
- 大小随机（1-3px）
- 随机分布在整个画布上
- 设置为背景层，不遮挡任何内容

### 7. 模块化设计原则
- 所有功能模块独立设计，可单独复用
- 统一组件库（按钮、卡片、输入框、标签等）
- 功能模块可自由组合，便于后续扩展
- 状态管理统一规范（加载、空状态、错误状态）

### 8. 用户便捷操作原则
- 一步完成原则：能在一步完成的绝不拆分为多步
- 智能预判：系统预判用户可能的操作，提供快捷入口
- 上下文延续：从一个操作到下一个，保持上下文信息
- 必要流程保留：核心流程（如创作）保留完整步骤，但可通过快捷入口快速体验

### 基础组件提示词

#### 按钮组件
```
在 Figma 中设计一套按钮组件，采用玻璃拟态科技风格。

设计内容：
1. 主要按钮（Primary Button）
   - 背景：主色 #3B82F6，带霓虹发光效果
   - 文字：白色 #FFFFFF
   - 圆角：8px
   - 状态：默认、Hover、点击、禁用
2. 次要按钮（Secondary Button）
   - 背景：透明，1px 主色描边
   - 文字：主色 #3B82F6
   - 圆角：8px
3. 文本按钮（Text Button）
   - 仅文字，无背景
   - 文字：主色 #3B82F6
4. 图标按钮（Icon Button）
   - 圆形玻璃效果
   - 圆角：50%
5. 按钮变体
   - 小尺寸（高度：32px）
   - 中尺寸（高度：40px）
   - 大尺寸（高度：48px）

设计风格：
- 玻璃拟态科技风格
- 所有按钮带轻微外发光效果
- Hover 状态有玻璃反光效果
- 点击状态透明度加深
```

## 📝 Figma 工作流程建议

### 1. 设计系统先行
- 先完成色彩、文字、图标系统
- 再建立基础组件库
- 最后开始页面设计

### 2. 从线框到高保真
- 先画简单的线框图确定布局
- 然后应用设计系统做高保真
- 最后添加交互和动效

### 3. 组件复用
- 所有相似元素都做成组件
- 使用组件变体（Variants）管理不同状态
- 建立组件库便于后续维护

### 4. 响应式设计
- 先设计移动端（375px）
- 再扩展到平板和桌面端
- 保持相同的视觉风格
