# 短视频创作者增强工具需求文档

## 1. AI 封面生成器 `cover_generator`

### 1.1 问题

短视频创作者每条视频都要做封面。当前流程：视频导出 → 打开醒图/Canva → 手动选帧截图 → 加标题文字 → 套模板 → 导出 → 回到发布页上传。每天重复 N 次。一条视频的封面制作时间往往比剪视频还长。

### 1.2 功能

```
视频 URL / 灵感库视频
  │
  ▼
① 智能选帧（FFmpeg，服务端，免费）
  - 按 1s 间隔抽取候选帧
  - 评分：人脸检测（居中、正面朝向、清晰度） + 画面复杂度 + 对比度
  - 返回 Top 3 候选帧，用户选择
  │
  ▼
② 标题生成（LLM）
  - 读视频转写文本 → 提取核心卖点
  - 生成 5 个封面标题候选（短、有冲击力、适合竖屏阅读）
  - 用户可选文案风格：悬念 / 数字 / 痛点 / 对比
  │
  ▼
③ 封面合成（服务端 Canvas/Satori，免费）
  - 选帧图 + 标题文字 → 叠加合成
  - 预设风格模板:
    · 大字报（竖排大字，高对比）
    · 上下分割（上图下标题）
    · 左右分割（左图右标题）
    · 居中贴纸（标题叠在图上，半透明底色）
  - 调参：字号、颜色、位置
  │
  ▼
④ 导出 1080×1920 (9:16) 或适配目标平台
  - 保存到灵感库
  - 关联到发布时作为封面
```

### 1.4 技术实现

| 步骤 | 依赖 | 说明 |
|------|------|------|
| 选帧 | FFmpeg `select` filter + `sharp` | 服务端，每秒抽帧，sharp 算清晰度/对比度 |
| 人脸评分 | 不需要接 API | 用 sharp 简单指标（对比度 + 色彩分布）即可，后期可加阿里云人脸 API |
| 标题生成 | `callDeepSeek()` | 已有 |
| 合成 | `sharp` (SVG/Canvas → PNG) 或 Satori | 服务端渲染文字叠图，不依赖前端 |
| 导出 | sharp 输出 PNG/JPEG | 已有 sharp |

无需新增外部 API。

## 2. AI 混剪 `auto_mashup`

### 2.1 问题

创作者手机里一堆零碎素材：产品特写、场景空镜、使用过程、B-roll… 想拼成一条快节奏推广视频，当前全靠手动在剪映拉时间轴：排顺序、调时长、加转场、卡 BGM 节奏点。5 条素材拼 30 秒视频手动做要 20-30 分钟。

### 2.2 功能

```
扔进去 5-20 段素材 + 一句目标描述
  │
  ▼
① 素材分析（服务端，免费）
  - 每段素材提取首帧缩略图
  - 检测时长、分辨率、画面内容（是否有人脸、产品、文字）
  │
  ▼
② 编排方案（LLM，~1000 token）
  - 根据目标（种草/开箱/教程/Vlog）生成镜头顺序
  - 每个镜头分配时长（重点素材多给时间）
  - 建议转场类型（硬切/淡入淡出/滑动）
  - BGM 风格推荐
  │
  ▼
③ 预览方案 → 用户调整
  - 时间轴预览（素材缩略图 + 时长条）
  - 可拖拽调顺序、拖边缘调时长
  - 可替换某段素材
  │
  ▼
④ 自动合成（服务端 FFmpeg）
  - 裁剪/填充 → 统一分辨率
  - 排 timeline → concat + 转场 xfade
  - 卡点 BGM（根据 beat 检测自动对齐）
  - 可选：自动字幕、片头片尾
  │
  ▼
⑤ 导出视频 → 保存到灵感库
```

### 2.4 技术实现

| 步骤 | 依赖 | 说明 |
|------|------|------|
| 素材分析 | FFmpeg `ffprobe` | 获取时长、分辨率、编码 |
| 缩略图 | FFmpeg 首帧提取 | 已有 |
| 编排方案 | `callDeepSeek()` | 根据素材元信息 + 目标生成镜头顺序 |
| BGM 卡点 | FFmpeg + librosa (可选) | 简单版：按 BPM 分配镜头切换点；复杂版：Python librosa beat detect |
| 合成 | FFmpeg concat + xfade | concat demuxer 硬切，xfade filter 做转场 |
| 分辨率统一 | FFmpeg scale + pad | 已有 `compose-video.ts` 里的逻辑 |

librosa beat detection 可选，初期用规则（固定 30s / BPM 120 → 每 2 秒切一个镜头），效果已经够用。

## 3. 标题优化器 `title_optimizer`

### 3.1 问题

同一段视频发抖音 vs B站，标题策略完全不同。抖音要 hook、要悬念、要短；B站要信息量、要关键词、要长。创作者通常随便写一个复制粘贴发全平台，效果打折扣。

### 3.2 功能

```
输入: 视频 URL 或粘贴视频转写文本

① 提取内容（免费）
  - 有视频 → FunASR 转写
  - 纯文本 → 直接分析

② LLM 分析（~800 token）
  - 内容类型（教程/种草/开箱/Vlog/新闻）
  - 核心卖点提取
  - 目标受众判断

③ 多平台标题生成
  - 抖音（≤30字）：hook + 悬念 + emoji
  - 小红书（≤20字）：关键词密集 + 痛点 + emoji
  - B站（≤80字）：信息量 + 关键词 + 标签
  - 视频号（≤30字）：温和 + 正能量 + 贴近生活
  - 快手（≤20字）：直白 + 接地气 + 老铁风格
  - YouTube（≤100字符）：SEO 关键词 + 英文

④ 每个平台 3-5 个候选，标注类型
  - 🔥 悬念型: "用了7天，我的皮肤变成这样…"
  - 📊 信息型: "精华液实测｜干皮星人的7天护肤记录"
  - 😱 情绪型: "熬夜党的救命精华液！"
  - 🎯 痛点型: "黄皮看过来！这个精华液真的能白"
  - 🛠️ 教程型: "30秒学会正确涂精华液"

⑤ 一键复制，或关联到发布页预填
```

## 4. 架构：页面 + Agent Tool 双入口

和智能编辑（smart-clip-requirements.md）一样，三个工具都共享底层引擎，同时提供页面和 Agent Tool 两种入口。

```
  cover-generator.ts / mashup-engine.ts / (LLM)
          │
  ┌───────┴───────┐
  │               │
  API 路由      Agent Tool
  │               │
  页面           对话调用
```

### 4.1 Agent 组合调用

三个工具作为 Agent Tool 注册后，可以被 Agent 自动编排：

```
用户: "把这段视频剪干净，做封面，写标题，发到抖音和小红书"
  │
  ▼
Agent 自动编排:
  ① smart_clip         → 去废话、去停顿
  ② cover_generator    → 智能选帧 + 生成封面
  ③ title_optimizer    → 生成抖音标题 + 小红书标题
  ④ publish            → 发布到抖音 + 小红书
```

```
用户: "把昨天直播回放做成10条带货短视频，全发"
  │
  ▼
Agent 自动编排:
  ① smart_slice        → 直播 → 10 条切片
  ② cover_generator ×10 → 每条切片封面
  ③ title_optimizer ×10 → 每条切片多平台标题
  ④ publish ×10        → 批量发布
```

### 4.2 单步调用

用户也可以只让 Agent 做其中一步：

```
"给这个视频做张封面"    → Agent 调 cover_generator
"帮我想5个抖音标题"     → Agent 调 title_optimizer
"把这几个素材混剪成30秒" → Agent 调 auto_mashup
```

## 6. 实现优先级

| 优先级 | 功能 | 工时 | 理由 |
|--------|------|------|------|
| P0 | AI 封面生成器 | 1.5 天 | 刚需，复用现有视频处理 + LLM，无新依赖 |

## 10. 上传限制

### 上传方式

同智能编辑（smart-clip-requirements.md §12.2）：客户端直传 Supabase Storage → API 收 URL → 服务端下载处理。不经过 Next.js body，绕过 30MB 限制。

### 并发 & 配额

| | 限制 |
|------|------|
| 混剪并发 | 2（共享 `mashup` 队列） |
| 封面/标题并发 | 无限制（轻量，不占 FFmpeg） |
| 月上传次数 | 复用现有配额 |

## 11. 双端适配说明

全部功能 Web 端 + 移动端 App 均可使用：

- 所有计算在服务端（FFmpeg / FunASR / LLM / sharp），前端仅展示和交互
- 上传：Web 拖拽/粘贴，移动端调原生相册（Capacitor `@capacitor/camera` / `@capacitor/filesystem`）
- 下载/保存：服务端写入 Supabase Storage，所有端统一 URL 访问
- 页面 UI：现有 TailwindCSS + mobile-first `max-w-[448px]`，H5 和 Capacitor WebView 一致体验
- 不依赖任何浏览器特有 API（如 WebGPU、WebAssembly、File System Access）

跨端无额外开发量。
