# 智能视频编辑需求文档

## 1. 概述

灵集智能视频编辑系统，提供两个方向的视频处理能力：

| 方向 | 功能 | 一句话 |
|------|------|--------|
| **智能剪辑** `smart_clip` | 去废话、去静音、去重复 → 精剪 | "把视频剪干净" |
| **智能切片** `smart_slice` | 提取高光/产品讲解/话题 → 多条短视频 | "把直播切成带货视频" |

两者共享底层引擎（音频提取 → ASR 转写 → 分析 → FFmpeg 执行），分析方向相反：剪辑找**废料**，切片找**高光**。

## 2. 核心场景

### 2.1 智能剪辑场景

| 场景 | 输入 | 输出 |
|------|------|------|
| 口播去废话 | 3 分钟口播（大量"嗯啊那个"、长停顿） | 1.5 分钟精剪版 |
| 静音切除 | 10 分钟直播回放 | 切除所有 >2s 的静音 |
| 删除指定段落 | 5 分钟录屏 + "把 2-3 分那段广告删了" | 指定区间删除，拼接无缝 |
| 去重复 | 教程视频（同一句话说了三遍） | 保留第一遍 |

### 2.2 智能切片场景

| 场景 | 输入 | 输出 |
|------|------|------|
| 直播带货切片 | 2 小时直播回放 | N 条 30-60s 带货短视频 |
| 产品讲解提取 | 1 小时产品发布会 | 每个产品的独立讲解视频 |
| 高能时刻捕获 | 游戏/赛事录播 | 高能片段合集 |
| 话题分段 | 30 分钟 Vlog | 按话题拆成 N 条独立视频 |

## 3. 架构决策：页面 + Agent Tool 双入口，共享引擎

### 3.1 分层架构

```
          ┌─────────────────────────┐
          │   smart-clip-engine.ts   │  ← 核心引擎（只写一次）
          │   提取/转写/分析/执行      │
          └───────────┬─────────────┘
                      │
        ┌─────────────┴─────────────┐
        │                           │
  ┌─────▼──────┐           ┌────────▼────────┐
  │  API 路由   │           │  Agent Tool     │
  │  /api/      │           │  ToolDefinition │
  │  smart-clip │           │  + handler      │
  └─────┬──────┘           └────────┬────────┘
        │                           │
  ┌─────▼──────┐           ┌────────▼────────┐
  │  独立页面   │           │  Agent 对话      │
  │  一步步操作  │           │  一句话调用      │
  └────────────┘           └─────────────────┘
```

引擎只写一次，页面和 Agent Tool 都是它的消费者。

### 3.2 两种入口，两种场景

| | 独立页面 | Agent 对话 |
|------|---------|-----------|
| 入口 | Agent 页面工具栏点击 | 聊天框输入需求 |
| 场景 | "我要精细控制每个 cut 点" | "帮我把这个视频剪干净" |
| 交互 | 分步 UI，预览调整 | 一句话 → 确认 → 执行 |
| 适合 | 重度用户、精细操作 | 轻度用户、批量自动化 |

### 3.3 Agent 组合调用

Agent 的真正价值是把多个工具串起来：

```
用户: "把这段直播剪成带货切片，每条加封面和标题，发到抖音"
  │
  ▼
Agent 自动编排:
  ① smart_slice          → 直播 → 5 条切片
  ② cover_generator (×5) → 每条切片生成封面
  ③ title_optimizer (×5) → 每条切片生成标题
  ④ publish (×5)         → 全部发布到抖音
```

没有 Agent，用户要手动跑 4 个页面、操作 16 次。Agent 一句话搞定。

## 6. API 设计

### 6.1 `POST /api/smart-clip/analyze`

```
请求:
{
  videoUrl: string,
  direction: 'clip' | 'slice',

  // clip 参数
  clipMode?: 'auto' | 'silence_only' | 'by_description' | 'by_time_ranges',
  description?: string,
  timeRanges?: Array<{start: number, end: number}>,

  // slice 参数
  sliceMode?: 'product' | 'highlight' | 'topic' | 'uniform' | 'custom',
  sliceDuration?: { min: number, max: number },   // 每段时长范围
  keywords?: string[],                             // 产品关键词（product 模式）
}

响应:
{ success: true, data: { taskId: string, streamUrl: string } }
```

### 6.2 `POST /api/smart-clip/execute`

```
请求:
{
  taskId: string,
  // clip 模式
  segments?: Array<{ start: number, end: number, action: 'keep' | 'cut' }>,
  // slice 模式
  slices?: Array<{ start: number, end: number, enabled: boolean, title?: string }>,
  // 后处理
  postProcess?: {
    ratio?: '9:16' | '16:9' | '1:1' | 'original',
    subtitles?: boolean,
    bgm?: string,
    endCard?: boolean,  // 片尾关注引导
  }
}

响应:
{ success: true, data: { taskId: string } }
```

### 6.3 `GET /api/smart-clip/stream?taskId=xxx`

SSE 流，推送分析/执行进度（同之前设计）。

## 7. 底层模块

### 7.1 共享引擎 `src/lib/ai/smart-clip-engine.ts`（~400 行）

```typescript
// 音频
extractAudio(videoPath: string): Promise<string>

// 转写
transcribeAudio(audioPath: string): Promise<TimedSentence[]>

// 检测（规则引擎，免费）
detectSilence(audioPath, threshold, minDuration): Promise<SilenceRange[]>
detectFillers(sentences: TimedSentence[]): FillerRange[]
detectRepetition(sentences: TimedSentence[]): RepetitionRange[]

// 分析（LLM，仅复杂模式）
analyzeForClip(sentences, description?): Promise<SegmentAnalysis[]>
analyzeForSlice(sentences, mode, keywords?): Promise<SlicePoint[]>

// 执行
executeClip(videoPath, segments, outputPath): Promise<string>
executeSlice(videoPath, slices, outputDir): Promise<string[]>
```

### 7.2 切片检测策略

| 切片模式 | 检测方法 | LLM | 精度 |
|---------|---------|-----|------|
| **带货切片** `product` | LLM 识别产品介绍句（"这是..."、"给大家看..."）+ 关键词匹配 | 需要 | 高 |
| **高能时刻** `highlight` | 音频能量检测（音量突增 +20dB）+ 语速突增 >50% | 不需要 | 中 |
| **话题分段** `topic` | LLM 语义分段 | 需要 | 高 |
| **时长均分** `uniform` | 直接按时间切，零分析 | 不需要 | — |
| **自定义** `custom` | 用户提供关键词或正则 | 不需要 | 取决于用户 |

带货切片和高能时刻是最常用的两个模式。

## 8. 并发控制

```typescript
// src/lib/jobs/queue.ts
CONCURRENCY_LIMITS: {
  smart_clip: 2,   // FFmpeg -threads 1
}
```

单任务限一核，2 并发占满 2 核 FFmpeg，剩余核留给 Next.js + FunASR。

## 9. Agent Tool 定义

页面开发完成后，每个功能各封装一个 Agent Tool。Tool handler 直接调用 engine 函数，和 API 路由走同一套底层代码。

### 9.1 `smart_clip` — 智能剪辑

```typescript
{
  name: 'smart_clip',
  isLongRunning: true,
  description: `智能视频剪辑。自动识别并切除废话、停顿、重复内容。
可以用自然语言描述想删什么（"去掉中间广告那段"），或指定精确时间区间。`,
  parameters: {
    videoUrl: { type: 'string', required: true },
    mode: { type: 'string', enum: ['auto', 'silence_only', 'by_description', 'by_time_ranges'] },
    description: { type: 'string' },  // mode=by_description 时必填
    timeRanges: { type: 'array', items: { start: 'number', end: 'number' } },
    silenceThreshold: { type: 'number', default: -30 },
    minSilenceDuration: { type: 'number', default: 2.0 },
  }
}
```

### 9.2 `smart_slice` — 智能切片

```typescript
{
  name: 'smart_slice',
  isLongRunning: true,
  description: `智能视频切片。从长视频（直播回放、发布会等）中自动提取高光片段，
生成多条短视频。支持带货切片、高能时刻、话题分段、时长均分等模式。`,
  parameters: {
    videoUrl: { type: 'string', required: true },
    sliceMode: { type: 'string', enum: ['product', 'highlight', 'topic', 'uniform', 'custom'] },
    keywords: { type: 'array', items: 'string' },  // product 模式的产品关键词
    sliceDuration: { type: 'object', properties: { min: 'number', max: 'number' } },
  }
}
```

### 9.3 Agent 调用示例

```
用户: "帮我把这个直播回放剪成带货切片"
Agent: → analyze_for_slice(video, mode='product')
       → 识别到 5 个产品讲解段落
       → "检测到 5 个可切片段落：精华液介绍(42s)、限时优惠(80s)... 确认执行吗？"
用户: "确认"
Agent: → execute_slice(video, slices)
       → 5 条切片生成完成，已保存到灵感库
```

## 12. 上传限制

### 12.1 单文件限制

| 功能 | 单文件上限 | 说明 |
|------|----------|------|
| 智能剪辑 | 500MB | 约 2 小时 1080p 口播/直播 |
| 智能切片 | 1GB | 直播回放可能更长 |
| AI 混剪 | 200MB/段 | 最多 20 段，总素材 ≤ 4GB |
| 封面生成 | 500MB | 和剪辑一致 |
| 标题优化 | 500MB | 只提取音频，不需要画质 |

### 12.2 绕过 30MB 瓶颈

当前 `next.config.mjs` 中 `serverActions.bodySizeLimit: '30mb'`。智能编辑入口不走 Server Action form 上传，改为：

```
客户端 ──直传──→ Supabase Storage (SDK)  → 返回 URL
                                                  │
客户端 ──POST JSON {videoUrl}──→ API ──下载──→ 服务端处理
```

API 只传 URL 字符串（几十字节），实际大文件走 Supabase Storage 直传，不经过 Next.js body。瓶颈转移到 Supabase 套餐上限（一般 50MB-5GB/文件）。

### 12.3 Nginx 配置

ECS 部署的 Nginx 需要放开 `client_max_body_size`：

```nginx
# 智能编辑 API 路由
location /api/smart-clip/ {
    client_max_body_size 0;  # 不限制（API 只收 URL JSON）
    proxy_pass http://127.0.0.1:3000;
}
```

Supabase Storage 直传走 Supabase 域名，不经过 ECS Nginx，不存在此限制。

### 12.4 数量限制

| | 限制 | 说明 |
|------|------|------|
| 智能剪辑 | 单次 1 段视频 | 剪辑对象是单段视频 |
| 智能切片 | 单次 1 段视频 | 切片对象是单段视频 |
| AI 混剪 | 单次 2-20 段素材 | 少于 2 段不需要混剪，多于 20 段编排复杂度过高 |
| 并发任务 | 2 个 | 共享 `smart_clip` 队列 |
| 月上传次数 | 复用现有配额（free 100/pro 1000/creator 10000） |
