## 一、产品定位

灵集是一款面向中文创作者的 AI 灵感收集 + 热点监控 + 内容创作工具。

用户可以通过文字、语音、链接、图片、视频快速记录灵感或收藏素材，系统自动提取内容、总结要点、生成标题、标签和分类，并沉淀到个人云端知识库。同时，用户可以配置热点关键词，系统自动从多个平台抓取和分析热点内容，通过 AI 进行真假识别、相关性分析、智能摘要，并实时推送通知。后续用户可以基于这些素材和热点生成小红书文案、短视频脚本、公众号大纲，并通过「去 AI 味」能力生成更自然、更像真人创作的内容。

**核心卖点：**

> 随手记灵感，自动抓热点，一键生成内容。

**首页文案方向：**

- 随手记灵感
- 收藏链接、上传截图和视频
- 配置关键词，AI 自动监控全网热点
- AI 自动总结归类
- 一键生成小红书、短视频、公众号内容
- 去 AI 味，像真人写的

**设计理念：简便，易上手。**

所有功能围绕「少一步操作就多一分使用」设计，减少用户认知负担，让记录灵感像发朋友圈一样简单。

**第一阶段定位：**

中文创作者的随身 AI 素材库 + 灵感库 + 写作助手。

## 二、目标用户

| 用户类型 | 典型场景 |
|---------|---------|
| 小红书博主 | 收藏爆款截图、链接，生成种草文案 |
| 短视频创作者 | 收藏视频素材，生成口播稿和分镜脚本 |
| 公众号作者 | 收藏文章链接，生成大纲和深度内容 |
| 自媒体运营 | 批量整理素材，快速产出多平台内容 |
| 文案策划 | 收集案例和灵感，生成创意文案 |
| 知识博主 | 整理学习笔记，输出知识内容 |

第一版以个人用户为主，每个用户拥有自己的私有灵感库。协作功能与个人功能同步开发。

## 三、首版开发形态

### 第一版只做

- Web App（Next.js，响应式设计）
- 移动端 H5（同一套代码，响应式适配）

### 暂不做

- iOS 原生 App
- Android 原生 App
- 微信小程序

### 原因

- Web + H5 可以最快验证主链路
- 避免一开始被多端开发拖慢
- 后续可以复用后端、数据库、AI 工作流
- H5 可先满足手机端使用场景

### 后续扩展顺序

1. Web App + 移动端 H5（第一版）
2. 微信小程序（第二阶段）
3. React Native iOS / Android App（第三阶段）
4. 浏览器插件（第三阶段）

## 四、MVP 核心目标

第一版验证两条主链路：

```
1. 记录/收藏素材 → AI 提取内容 → AI 总结归类 → 保存到个人灵感库 → 可搜索、可复用、可生成文案
2. 配置监控关键词 → 定时从多平台抓取 → AI 真假识别/相关性分析 → 智能摘要 → 实时推送通知 → 保存到热点库 → 可用于创作
```

### MVP 必做功能

- 文字记录灵感
- 语音记录灵感
- 链接解析并提取文案
- 图片上传并总结归类
- 视频上传并总结归类
- AI 自动标题、摘要、标签、分类
- 灵感库列表和详情页
- 搜索与筛选
- AI 生成小红书/短视频/公众号内容（图文并茂，配图由 Seedance 自动生成并排版）
- 去 AI 味生成
- 协作与分享
- 热点监控（配置关键词、多平台抓取、AI 分析、实时通知）
- 订阅与用量限制

## 五、系统架构

### 5.1 架构总览

```
┌─────────────────────────────────────────────────────┐
│                    客户端层                          │
│  ┌──────────────────────────────────────────────┐   │
│  │  Web App + H5（Next.js，响应式，统一代码库）   │   │
│  └──────────────────────┬───────────────────────┘   │
└─────────────────────────┼───────────────────────────┘
                          │
┌─────────────────────────┼───────────────────────────┐
│                    服务层                             │
│  ┌──────────┐  ┌───────┴───────┐  ┌──────────────┐ │
│  │ Next.js   │  │  API Routes   │  │  Edge Funcs  │ │
│  │ SSR/SSG   │  │  (BFF 层)     │  │  (定时任务)   │ │
│  └──────────┘  └───────┬───────┘  └──────────────┘ │
└────────────────────────┼────────────────────────────┘
                         │
┌────────────────────────┼────────────────────────────┐
│                   数据与 AI 层                        │
│  ┌──────────┐  ┌──────┴──────┐  ┌────────────────┐ │
│  │ Supabase │  │  阿里云 OSS  │  │  AI 服务集群    │ │
│  │ Auth/DB/ │  │  (素材存储)  │  │ DeepSeek/豆包/  │ │
│  │ Realtime │  │             │  │ Seedance        │ │
│  └──────────┘  └─────────────┘  └────────────────┘ │
└─────────────────────────────────────────────────────┘
```

### 5.2 核心设计原则

1. **BFF 层（Backend For Frontend）**：Next.js API Routes 作为统一入口，客户端不直接调用第三方 AI 服务，所有请求经 BFF 层转发，保护 API Key 安全
2. **Supabase 三合一**：Auth（认证）+ PostgreSQL（数据）+ Realtime（实时同步），减少自建后端工作量
3. **素材存储分离**：结构化数据存 Supabase，媒体文件（图片/视频/音频）存阿里云 OSS
4. **异步处理**：链接提取、视频理解等耗时操作走异步队列，前端轮询或 Supabase Realtime 推送结果
5. **降级策略**：DeepSeek 主用，失败自动降级到 `deepseek-v4-flash`；豆包失败降级到系统原生能力

### 5.3 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Next.js + React + Tailwind CSS | 响应式，Web + H5 统一代码库 |
| 后端 BFF | Next.js API Routes | 统一 API 入口，保护 API Key |
| 数据库 | Supabase (PostgreSQL) | Auth + DB + Realtime + pgvector |
| 缓存/队列 | Supabase + BullMQ | 用量计数缓存 + 异步任务队列 |
| 对象存储 | 阿里云 OSS | 媒体文件存储 |
| 文本 AI | DeepSeek | 总结、归类、写作、关联推荐 |
| 多模态 AI | 豆包 | 图片理解、视频理解、OCR |
| 语音识别 | 分级方案 | 免费用户系统原生 ASR，付费用户阿里云 STT |
| 图片生成 | Seedance | AI 生图 |
| 视频生成 | Seedance | AI 生视频 |
| 向量搜索 | PostgreSQL pgvector | 语义搜索 |
| 支付 | 微信支付 | Web/H5 端支付 |

### 5.4 AI 调用架构

```
客户端 ──→ BFF(Next.js API Routes) ──→ AI 服务
                │
                ├── /api/ai/summarize      → DeepSeek（单条/聚合总结）
                ├── /api/ai/categorize     → DeepSeek（自动归类）
                ├── /api/ai/relate         → DeepSeek（关联推荐）
                ├── /api/ai/transform      → DeepSeek（灵感转内容）
                ├── /api/ai/deaiify        → DeepSeek（去 AI 味改写）
                ├── /api/ai/extract-link   → DeepSeek + 豆包（链接提取）
                ├── /api/ai/understand     → 豆包（图片/视频理解）
                ├── /api/ai/transcribe     → 阿里云 STT（高精度语音识别）
                ├── /api/ai/generate-img   → Seedance（AI 生图）
                └── /api/ai/generate-vid   → Seedance（AI 生视频）
```

## 六、模块化架构设计

### 6.1 设计原则

#### 6.1.1 用户操作优先原则
- **一步完成原则**：能在一步操作中完成的任务绝不拆分为多步
- **必要流程保留原则**：某些必须的流程（如登录、支付、AI生成）步骤保持完整，但可优化体验
- **智能预判原则**：系统预判用户可能的操作，提供快捷入口
- **上下文延续原则**：从一个操作到下一个操作，保持上下文信息延续

#### 6.1.2 模块化架构原则
- **松耦合高内聚**：各模块独立开发、独立部署、独立测试
- **可插拔架构**：模块支持热插拔，便于扩展和替换
- **可组合性**：模块可自由组合形成新功能
- **标准化接口**：模块间通过标准化接口通信

### 6.4 模块化设计原则

#### 6.4.1 组件可复用性
- **原子组件**：Button、Input、Card等基础组件全局复用
- **复合组件**：由多个原子组件组成的功能组件，在对应模块内复用
- **页面组件**：完整页面组件，由复合组件和原子组件组成

#### 6.4.2 状态管理
- **全局状态**：用户信息、订阅状态、通知中心状态（使用 Zustand）
- **页面状态**：页面内的临时状态（使用 React useState/useReducer）
- **数据缓存**：使用 React Query/SWR 进行数据缓存和管理

#### 6.4.3 样式管理
- **设计系统**：统一的颜色、字体、间距、圆角等规范
- **CSS-in-JS**：使用 Tailwind CSS + shadcn/ui 实现样式模块化
- **主题支持**：支持深色/浅色模式切换

#### 6.4.4 路由设计
- **嵌套路由**：支持模块间的嵌套路由
- **动态路由**：支持内容详情页的动态路由
- **路由懒加载**：页面级组件懒加载，优化性能

### 6.5 用户操作优化设计

#### 6.5.1 首页快捷入口优化
**原流程**：
```
点击功能 → 跳转页面 → 填写内容 → 保存 → 返回首页
```
**优化后（一步操作优先）**：
```
首页悬浮快速记录栏 → 输入内容/上传媒体 → 一键保存（自动AI总结）
```
同时保留完整页面入口供深度操作。

#### 6.5.2 AI创作流程优化
**原流程**：
```
AI写作页 → 选择素材 → 选择类型 → 选择风格 → 生成 → 编辑 → 保存
```
**优化后**：
```
保留完整流程（必要流程），但提供一键生成快捷选项：
- 一键生成小红书文案（使用最近3条灵感+默认风格）
- 一键生成短视频（使用最近热点+默认模板）
- 智能推荐生成类型（基于用户习惯）
```

#### 6.5.3 灵感/热点详情页优化
**设计原则**：
- 上下文延续：从列表到详情，保持操作上下文
- 快捷操作：详情页提供一键转素材、一键生成等快捷按钮
- 相关推荐：展示相关灵感/热点，减少切换

#### 6.5.4 搜索体验优化
**设计原则**：
- 全局搜索：一键呼出，支持灵感+热点同时搜索
- 实时搜索：输入即出结果，无需点击搜索按钮
- 搜索历史：保留搜索历史，一键复用
- 智能建议：搜索过程中提供智能建议

### 6.6 可扩展性设计

#### 6.6.1 功能模块扩展
- **插件化架构**：支持动态加载新功能模块
- **市场集成**：未来可集成第三方插件市场

#### 6.6.3 数据源扩展

## 七、核心功能需求

### 7.1 用户系统

**登录方式：**

| 方式 | 说明 | 阶段 |
|------|------|------|
| 手机号 + 验证码 | 注册和登录 | 第一版 |
| 手机号 + 密码 | 登录 | 第一版 |
| 微信扫码登录 | Web 端 | 第一版 |
| 微信内 H5 授权 | 微信浏览器内 | 第一版 |

**注册流程：**

1. 输入手机号 → 发送验证码（60 秒间隔，5 分钟有效，每日上限 10 条）
2. 验证通过 → 设置用户名 + 密码（8 位以上，含字母+数字）
3. 注册完成 → 进入首页

**登录方式：**

- 手机号 + 密码
- 手机号 + 验证码

**用户数据：**

| 字段 | 说明 |
|------|------|
| id (uuid) | 主键 |
| phone | 手机号 |
| username | 用户名 |
| password_hash | bcrypt 哈希 |
| avatar_url | 头像 |
| wechat_openid | 微信 OpenID |
| wechat_unionid | 微信 UnionID |
| plan | 订阅等级（free/pro/creator） |
| usage | 用量统计 |
| created_at | 注册时间 |
| updated_at | 更新时间 |

**安全要求：**

- 每个用户数据隔离，只能访问自己的灵感库
- 支持用户注销和删除个人数据
- JWT Token：Access Token 1 小时过期，Refresh Token 30 天
- 支持查看已登录设备，远程登出

### 7.2 文字记录

用户可以快速输入灵感。

**功能：**

- 输入标题（可选）
- 输入正文
- 一键保存
- 保存后 AI 自动总结
- 用户可编辑 AI 生成结果

**AI 输出：**

| 输出项 | 说明 |
|--------|------|
| 自动标题 | 基于内容生成简洁标题 |
| 一句话摘要 | 核心要点概括 |
| 详细摘要 | 完整内容总结 |
| 关键要点 | 提取 3-5 个关键点 |
| 标签 | 自动生成 3-8 个标签 |
| 分类 | 自动归入预设分类 |
| 可生成内容方向 | 建议可延展的创作方向 |

## 十二、数据管理

### 11.1 离线与同步

```
本地 IndexedDB 缓存
    │
    ├── 写入：灵感先存本地，标记为 pending
    ├── 同步：网络恢复后，按时间戳顺序同步到 Supabase
    ├── 冲突：最后修改时间优先，保留冲突版本供用户选择
    └── 缓存策略：最近 200 条灵感 + 所有分类和标签
```

## 十五、风险与注意事项

| 风险 | 应对策略 |
|------|---------|
| 链接解析被反爬 | 降级方案：手动粘贴+截图上传 |
| AI 成本失控 | 严格用量限制 + 防亏损规则 + 加量包 |
| 视频存储成本高 | 保留期限制 + 可选删除原视频 |
| 内容版权风险 | 定位为个人收藏+创作辅助，不鼓励搬运 |
| 敏感内容风险 | AI 输出需经内容安全过滤 |
| 早期用户少、无规模优势 | 定价宁高勿低，年付锁定用户 |
| 平台规则变化 | 链接解析遵守目标平台规则，降级方案兜底 |
| 用户隐私保护 | 系统监控词词库只统计匿名使用数据，不关联具体用户，所有用户监控词数据严格隔离独立 |
